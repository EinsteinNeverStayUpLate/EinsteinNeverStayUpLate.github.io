Spring 笔记

Spring

Spring框架优点如下：

  • 非侵入式设计（non-invasive），使应用程序代码对框架的依赖最小化。
  • 方便解耦，简化开发，将对象的创建和依赖关系的维护交给Spring容器管理
  • 支持AOP
  • 支持声明式事务处理：通过配置就可以完成对事务的支持，无需编程
  • 方便测试
  • 方便集成各种优秀框架
  • 降低API使用难度

控制反转(IoC)

​ 对象的实例不再由调用者来创建，而是由Spring容器来创建，Spring容器负责控制程序之间的关系，而不是由调用者的程序代码之间控制。这样，从调用者的角度来看，控制权由应用代码转移到了Spring容器，控制权发生了反转。

依赖注入（DI）

​ 从Spring容器角度来看，Spring容器负责将依赖对象赋值给调用者的成员变量，这相当于为调用者注入了它依赖的实例，这就是Spring的依赖注入。

Bean的装配方式

基于XML的装配

  • 构造注入（Constructor Injection）Bean的类中必须提供一个带所有参数的有参构造方法

  • 设值注入（Setter Injection）Bean类中必须包含需要注入属性的对应的setter方法，其提供一个无参构造器。

SpringAOP

AOP：Aspect-Oriented Programming，即面向切面编程。它是面向对象编程（OOP）的一种补充，目前是一种比较成熟的编程方法。

AOP术语

  • Aspect（切面）：指封装的用于横向插入的系统功能（如事务、日志等）的类。
  • Advice（通知/增强处理）：AOP框架在特定的切入点执行的增强处理，即在定义好的切入点处所要执行的程序代码。可以将其理解为切面中的方法，它是切面的具体实现。
  • Target Object（目标对象）：是指所有被通知的对象，也称为被增强对象。如果AOP框架采用的是动态AOP实现，那么该对象就是一个被代理对象。
  • Proxy（ 代理）：将Advice应用到目标对象之后，被动态创建的对象。
  • Weaving（织入）：将切面代码插入到目标对象上，从而生成代理对象的过程。

CGLIB代理（Code Generation Library）

JDK动态代理非常简单，但它还有一定的局限性——使用动态代理的对象必须实现一个或多个接口。如果有对没有实现接口的类进行处理，那么可以使用CGLIB代理。

引入：CGLIB（Code Generation Library）是一个高性能开元的代码生成包，它采用非常底层的字节码技术，对指定的目标类生成一个子类，并对子类进行增强。
注意：从框架的模式来说，CGLIB和JDK动态代理的逻辑上没有太大区别

Spring的通知类型

  • 环绕通知：在目标方法执行前后实施增强，可以应用于日志、事务管理等功能。
  • 前置通知：在目标方法执行前实施增强，可以应用于权限管理等功能。
  • 后置通知：在目标方法执行后实施增强，可以应用于关闭流、上传文件、删除临时文件等功能
  • 异常通知：在方法抛出异常后实施增强，可以应用于处理异常记录日志等功能。
  • 引介通知：在目标类中添加一些新的方法和属性，可以应用于修改老版本程序（增强类）。
