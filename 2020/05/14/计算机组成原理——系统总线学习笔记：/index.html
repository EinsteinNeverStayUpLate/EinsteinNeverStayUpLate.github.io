计算机组成原理——系统总线学习笔记

计算机部件之间的连接方式：

  • 各部件之间使用单独的连线
  • 将各部件连到一组公共信息传输线上，称为总线（Bus）

什么是总线？

总线是连接多个部件的信息传输线，是各部件共享的传输介质。

组成：

由许多传输线或通路组成，每条线可一位一位地传输二进制代码。

总线结构：

  • 面向CPU的双总线结构
  • 单总线结构
  • 以存储器为中心的双总线结构

总线分类：

  • 片内总线（如CPU芯片内部，寄存器之间，寄存器与ALU之间的连线）
  • 系统总线：指CPU，主存，I/O设备（通过I/O接口）各大部件之间的信息传输线。
    • 数据总线：是双向传输总线，其位数与机器字长、存储字长有关
    • 地址总线：其位数与MAR位数相同
    • 控制总线
  • 通信总线：用于计算机系统之间或计算机系统与其他系统之间的通信。

What is DMA?

DMA(direct memory access)is a hardware feature that allows memory access to occur independently of the program currently run by the micro processor. It can either be used by I/O devices to directly read from or write to memory without executing any micro processor instructions. Or, it can be used to efficiently copy blocks of memory. During DMA transfers,the micro processor can execute and unrelated program at the same time.

总线特性：

  • 机械特性：如插头与插座使用的标准
  • 电器特性：每一根传输线上信号的传递方向和有效地电平范围
  • 功能特性：总线每根传输线的功能
  • 时间特性：什么时候有效

总线性能指标：

  • 总线宽度：其实就是数据总线的根数
  • 总想带宽：即总线的数据传输速率
  • 时钟同步/异步 ：总线上的数据与时钟同步工作的总线称为同步总线，与时钟不同步工作的总线称为异步总线
  • 总线复用：总线的共用
  • 信号线数：地址总线，数据总线和控制总线三种总线数的总和
  • 总线控制方式
  • 其他指标

奇偶校验（Parity Check）：一种校验代码传输正确性的方法。根据被传输的一组二进制代码的数位中“1”的个数是奇数还是偶数来进行校验，采用奇数校验则称为奇校验，反之偶校验。采用何种校验方式是提前约定的。

总线标准： 可视为系统与各模块、模块与模块之间的一个互连的标准界面。

总线结构： 通常分为单总线结构和多总线结构两种。

总线控制：

​ 总线上连接着多个部件，什么时候由哪个部件发送信息，如何给信息传送定时，如何防止信息丢失，如何避免多个部件同时发送，如何规定接收信息的部件等一系列问题都需要由总线控制器统一控制。主要包括判优控制（或称逻辑仲裁）和通信控制。

  • 判优控制：包括集中式和分布式

​ 集中式包括：链式查询；计数器定时查询；独立请求方式。

  • 通信控制：主要解决通信双方如何获知传输开始和传输结束，以及通信双方如何协调如何配合。you四种方式：
    • 同步通信
    • 异步通信
    • 半同步通信
    • 分离式通信

通常将完成一次总线操作的时间称为总线周期，可分为以下四个阶段：

  • 申请分配阶段：由需要使用总线的主模块提出申请，经总线仲裁机构决定下一传输周期的总线使用权授予某一申请者。
  • 寻址阶段：取得了使用权的主模块通过总线发出本次要访问的从模块的地址及有关命令，启动参与本次传输的从模块。
  • 传数阶段：主模块和从模块进行数据交换，数据由原模块发出，经数据总线流入目的模块。
  • 结束阶段：主模块的有关信息均从系统总线上撤除，让出总线使用权。

同步通信：优点是，规定明确、统一，模块间的配合简单一致。缺点是，两模块同步的速度必须按较慢的一方，故降低了总线的工作效率。

异步通信：采用应答方式（又称为握手方式），当主模块发出Request信号后，一直等待从模块反馈回来Acknowledge信号，才开始通信。

  • 不互锁方式
  • 半互锁方式
  • 全互锁方式

异步串行通信的数据传送速率用波特率来衡量。波特率指单位时间内传送二进制的位数（包括起始低电平位，奇偶校验位，和终止高电平位），单位用bps（位/秒）表示，记为波特。

若只考虑有效数据位，可用比特率来衡量异步串行通信的数据传输速率。

半同步通信：

在同步通信的基础上加入一条“等待”响应信号线，采用插入时钟（等待）周期的措施来协调通信双方的配合问题。以输入为例，主模块在T1 发出地址，在T2发出命令，在T3传输数据，T4结束传输。如果从模块在T3到来无法提供数据，则必须在T3到来前通知主模块，给出Wait信号（低电平）。主模块在T3到来时刻测得WAIT为低电平，则等待一个时钟周期，结束后再测响应信号线，若还是低电平，再等待。反之则进行数据的传输。

分离式通信：

进一步分析读命令传输周期，发现除了申请总线这一阶段外，其余时间主要花费在如下3个方面：

  1. 主模块通过传输总线向从模块发送地址和命令
  2. 从模块按照命令进行读数据的必要准备
  3. 从模块经数据总线向主模块提供数据

这里注意到，2中是不需要用到总线的，但是前几种通信方式都是这一时刻占用着总线，故可以充分利用这一阶段的总线空闲时间。

思路是：将一个传输周期分解为两个子周期。在第一个子周期中，主模块A在获得总线使用权后将命令、地址以及其他有关信息，包括该主模块的编号发到系统总线上，经总线传输后，由有关的从模块B接受下来。然后从模块A放弃对总线的使用权。供其他模块使用。然后模块B根据A发送过来的各种信息，经选择，译码，读取等一些列内部操作，将A模块所需的数据准备好，再有B申请使用总线。
显然在这种情况下，上面的每个模块都变成了主模块。
